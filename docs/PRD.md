# 两轮电动车电池模拟器 PRD

## 1. 文档信息

- 产品名称：两轮电动车电池模拟器
- 当前版本：MVP v0.1
- 更新时间：2026-06-02
- 文档状态：执行中

## 2. 背景与目标

面向产品、研发和测试，提供一个网页版可视化工具，用于快速模拟两轮电动车电池包在不同工况下的电气输出表现。当前优先目标是搭建一个可运行的简单页面，能够展示核心输入参数与输出指标之间的联动关系，为后续更精细的电池模型、故障模拟和数据回放做基础。

## 3. 当前版本范围

### 3.1 已实现范围

- 单页 Web 界面，支持直接在浏览器内调节参数。
- 可调参数：动力输出目标、BMS 采样率、标称电压、BMS 限流阈值、等效内阻、油门开度。
- 实时输出指标：实际输出功率、实际电压、实际电流、功率缺口、热损耗、剩余电量、BMS 延迟。
- 最近 8 个采样点记录展示。
- 支持暂停采样、继续采样、重置场景。

### 3.2 当前不做

- 不做真实电芯级模型。
- 不做复杂温升模型与热扩散仿真。
- 不做车辆速度、坡度、轮胎阻力等整车动力学仿真。
- 不做历史数据导入导出。

## 4. 用户角色

- 产品经理：快速验证某个需求参数调整是否会导致输出指标变化。
- 前端/后端研发：对齐仿真展示结构和后续接口字段。
- 测试工程师：构造不同边界值场景，观察系统响应是否符合预期。

## 5. 核心需求

### 5.1 参数输入

- 用户可以通过滑杆与数值输入框修改关键仿真参数。
- 参数修改后，页面应立即反映对当前输出结果的影响。

### 5.2 实时仿真

- 页面以 BMS 采样率作为刷新依据更新仿真结果。
- 仿真结果至少包括：动力请求、实际输出、电压、电流、电量、延迟。

### 5.3 结果展示

- 页面需突出显示关键指标和风险状态。
- 页面需保留最近若干采样记录，便于观察变化趋势。

### 5.4 场景控制

- 用户可以暂停和恢复实时仿真。
- 用户可以一键重置到当前参数下的初始状态。

## 6. MVP 建模假设

- 动力请求由目标功率与油门开度共同决定。
- BMS 采样率影响响应效率和采样滞后，采样越低，输出折损越明显。
- 等效内阻造成压降与热损耗，热损耗与电流平方成正相关。
- 电量按固定电池容量近似衰减，用于展示趋势而非精确 SOC 算法。

## 7. 交互与体验要求

- 页面优先桌面端可用，同时在移动端保持基本可读性。
- 用户无需跳转页面即可完成参数配置与结果查看。
- 视觉上应清晰区分输入区域、实时状态区域和采样记录区域。

## 8. 非功能要求

- 本地运行简单，默认通过 Vite 开发环境启动。
- 代码结构保持轻量，便于后续替换为更真实的仿真模型。
- 允许在后续版本中接入图表组件和外部数据源。

## 9. 后续迭代方向

- 增加温度、单体压差、SOC 校准、均衡状态等 BMS 变量。
- 增加波形图或折线图，展示电压、电流和功率随时间变化。
- 增加典型场景模板，例如起步、爬坡、急加速、低电量返程。
- 增加故障注入，例如过流、欠压、采样异常、传感器漂移。

## 10. 需求流程约定

- 从本版本开始，所有新增需求必须先更新本 PRD，再进入实现。
- 每次功能完成后，再进行 git 提交。
- 若需求与当前 PRD 冲突，应先修改 PRD 中的范围、假设或验收标准。
