# 两轮电动车电池模拟器 PRD

## 1. 文档信息

- 产品名称：两轮电动车电池模拟器
- 当前版本：MVP v0.4
- 更新时间：2026-06-06
- 文档状态：执行中

## 2. 背景与目标

面向产品、研发和测试，提供一个网页版可视化工具，用于快速模拟两轮电动车电池包在不同工况下的电气输出表现。当前优先目标是在现有可运行页面基础上，引入真实电芯级近似建模与独立电池配置界面，能够区分 BMS 参数与电芯参数，并展示这些配置如何共同影响输出电压、电流、功率和电量状态；同时引入基于种子的地形生成功能，模拟真实路况（坡度、红绿灯）对电池放电工况的影响，为后续更精细的故障模拟和数据回放做基础。

## 3. 当前版本范围

### 3.1 已实现范围

- 单页 Web 界面，支持直接在浏览器内调节参数。
- 运行场景参数支持：动力输出目标、油门开度、电芯温度。
- 新增电池配置页面，配置区分为 BMS 和电芯两部分。
- BMS 配置支持：采样率、限流阈值、过充电压、截止电压、BMS OCV 曲线。
- 电芯配置支持：串并联节数、单芯容量、真实电芯 OCV 曲线、温度内阻曲线。
- 实时输出指标支持：实际输出功率、整包电压、实际电流、功率缺口、热损耗、剩余电量、采样延迟、真实单体 OCV、BMS 估算 OCV、单体负载电压、估算单体电压。
- 最近 8 个采样点记录展示。
- 支持暂停采样、继续采样、重置场景。

### 3.2 当前不做

- 不做按单颗电芯逐个离散仿真，仅做等效串并联近似。
- 不做复杂温升模型与热扩散仿真。
- 不做车辆速度、轮胎阻力等整车动力学仿真（坡度仿真见 3.3 节）。
- 不做历史数据的完整原始分辨率导出（当前暂存内存，后续支持秒级采样导出）。

### 3.3 路况地形生成（种子驱动）

基于确定性种子生成地形数据，用于模拟真实路况对电池放电工况的影响。

#### 3.3.1 种子与地形生成

- 用户可输入一个种子值（数字或字符串），用于驱动地形的伪随机生成。
- 种子相同且所有生成参数相同时，必须产出完全相同的地形序列，保证仿真可复现。
- 地形数据为沿行驶方向的连续高度序列，用于推导坡度、加减速需求等工况。

#### 3.3.2 骑行速度

- 用户可手动设置骑行平均速度（km/h），作为仿真中车辆行驶的基础速度。
- 上坡、下坡、起步以及正常行驶阶段的目标速度均以设定的平均速度为基准。
- 速度参数直接影响行驶时间与电池放电持续时长，是连接地形工况与电量消耗的关键输入。

#### 3.3.3 红绿灯

- 地形中沿行驶方向随机分布红绿灯停靠点。
- 用户可通过参数（红绿灯密度）控制单位距离内红绿灯的平均数量。
- 红绿灯位置、红灯/绿灯时长均由种子确定性生成。
- 遇到红灯时车辆需停车等待，绿灯可通过，影响电池放电节奏。

#### 3.3.4 上下坡与地形陡峭度

- 地形中随机生成上下坡段落，坡度由种子与陡峭度参数共同决定。
- 用户可通过「陡峭度」参数控制地形起伏的剧烈程度：值越大地形越陡峭，值越小越平坦。
- 用户可通过「起点海拔」与「终点海拔」参数设定整段路线的起止海拔差，用于修正地形整体的上下坡趋势。
  - 终点海拔高于起点：整体偏上坡，电池放电负载更高。
  - 终点海拔低于起点：整体偏下坡，负载相对更低。
  - 起终海拔相同：无整体趋势，纯随机起伏。
- 海拔差修正应在保持局部随机性的同时，使地形总高度变化量趋近于设定的海拔差值。

#### 3.3.5 确定性保证

- 在种子值、骑行平均速度、红绿灯密度、陡峭度、起点海拔、终点海拔全部相同的条件下，每次生成的地形序列、红绿灯分布必须完全一致。
- 实现上应使用种子化的伪随机数生成器（PRNG），避免引入时间戳、`Math.random()` 等非确定性源。

### 3.4 自定义剩余里程算法（C 语言函数）

用户可通过代码编辑器输入自定义的 C 语言函数，用于在仿真过程中实时计算预估剩余里程，替代内置的固定算法。

#### 3.4.1 函数接口

- 页面提供一个 C 语言代码输入框，用户在其中编写函数体。
- 函数签名固定，用户不得修改函数名与参数列表；参数为扁平类型，便于 WASM 线性内存直接传递。
- 固定签名：
  ```c
  float estimate_remaining_km(
      float pack_voltage,   // 包电压 (V)
      float current,        // 电流 (A)
      float soc,            // SOC (0.0~1.0)
      float cell_voltage,   // 单体电压 (V)
      float temperature,    // 电芯温度 (°C)
      float total_km,       // 总里程 (km)
      float trip_km,        // 小计里程 (km)
      unsigned long time_ms // 当前仿真时间 (ms)
  );
  ```
- 返回值：**预估剩余里程**（float，单位 km）。
- 函数体内允许使用 `static` 局部变量和全局变量，用于在多次调用之间保持内部状态（如历史采样队列、滑动平均值、趋势判断等）。
- 每次重新编译加载时，static/全局变量重置为初始值；同一份已编译的 WASM 模块实例内，状态跨调用持久保持。

#### 3.4.2 调用时机与仿真引擎

- 仿真基础步长默认为 **0.01 秒/点**（100Hz），用户可通过引擎配置面板修改精度（如 0.001s、0.01s、0.1s 等）。
- 每个仿真点均进行完整物理计算（电压、电流、SOC、里程更新等）。
- 用户自定义函数按 BMS 配置中的**采样频率**调用，而非每个仿真点都调用；两次调用之间的剩余里程保持上次返回值不变。
- 仿真运行速度由电脑性能自动决定，采用**能跑多快跑多快**的策略，不做人为限速；但每个仿真点的计算必须严格按设定步长推进，保证结果精度不受运行速度影响。

#### 3.4.3 数据采集与记录

- 仿真过程中，每个仿真点均需采集并记录完整的状态数据，包括但不限于：时间戳、包电压、电流、SOC、功率、单体电压、温度、总里程、小计里程、剩余里程（自定义函数返回值）、当前坡度、当前速度等。
- 记录数据按时间顺序存储，仿真结束后可浏览完整的历史数据序列。
- 当前版本数据暂存于内存，后续版本支持导出为秒级采样的表格数据（CSV/Excel 格式）。

#### 3.4.4 编辑与验证

- 代码输入框应支持基本的语法高亮与行号显示。
- 用户提交代码后，前端将函数体拼接为完整 C 源文件（含固定签名头文件），发送至后端编译服务。
- 后端使用 **Emscripten (emcc)** 将 C 源码编译为 WebAssembly (.wasm) 模块，编译参数：
  - `-O2` 优化、`-nostdlib`（不链接完整 libc）、`--no-entry`（无 main 函数）
  - `EXPORTED_FUNCTIONS` 仅暴露 `estimate_remaining_km`，其余符号全隔离
  - `STANDALONE_WASM=1`，不注入 JS 胶水代码，前端直接 `WebAssembly.instantiate` 加载
- 编译成功后，.wasm 模块返回前端并缓存；同一份代码不重复编译。
- 编译失败时，页面展示 emcc 返回的完整错误信息，仿真不予启动。
- 前端每次 `instantiate` 获得独立的 WASM 实例，实例间内存完全隔离，不会出现跨用户/跨次仿真的状态污染。
- 仿真运行过程中可实时观察自定义函数返回的剩余里程变化曲线。

## 4. 用户角色

- 前端/后端研发：对齐仿真展示结构和后续接口字段。

## 5. 核心需求

### 5.1 参数输入

- 用户可以通过滑杆与数值输入框修改运行场景参数。
- 用户可以进入电池配置页面，并在页面内切换或查看 BMS 与电芯两个配置分区。
- 用户可以直接编辑 OCV 曲线和温度内阻曲线的关键点数据。
- 引擎配置面板支持修改仿真精度（步长），默认 0.01 秒。
- 参数修改后，页面应立即反映对当前输出结果的影响。

### 5.2 实时仿真

- 仿真基础步长默认 0.01 秒/点（100Hz），可通过引擎配置面板修改。
- 每个仿真点进行完整物理计算，结果精度不受运行速度影响。
- 每个仿真点均采集完整状态数据并按时间序列记录，仿真结束后可浏览历史数据。
- 页面以 BMS 采样率作为刷新依据更新仿真结果。
- 仿真需同时考虑 BMS 截止阈值、过充阈值、BMS OCV 标定曲线与真实电芯 OCV、温度内阻曲线。
- 仿真结果至少包括：动力请求、实际输出、电压、电流、电量、延迟。
- 仿真需输出当前温度条件下的等效内阻与单体电压估算结果。

### 5.3 结果展示

- 页面需突出显示关键指标和风险状态。
- 页面需保留最近若干采样记录，便于观察变化趋势。
- 页面需显示 BMS 配置摘要和电芯配置摘要，便于验证当前使用的模型参数。

### 5.4 场景控制

- 用户可以暂停和恢复实时仿真。
- 用户可以一键重置到当前参数下的初始状态。

### 5.5 电池配置页面

- 页面应存在独立的电池配置区域，不与实时状态区混淆。
- 配置区域内分为 BMS 和电芯两个面板。
- BMS 面板至少包含：采样率、过充电压、截止电压、OCV 曲线。
- 电芯面板至少包含：真实电芯 OCV 曲线、温度内阻曲线。

## 6. MVP 建模假设

- 动力请求由目标功率与油门开度共同决定。
- BMS 采样率影响响应效率和采样滞后，采样越低，输出折损越明显。
- 电池包电压由单体 OCV 曲线按照当前 SOC 插值后乘以串联节数得到。
- 电芯内阻由温度内阻曲线按照当前温度插值估算，再换算为整包等效内阻。
- BMS OCV 曲线用于模拟 BMS 侧对电量估算与电压判断的标定曲线，可与真实电芯 OCV 曲线存在偏差。
- 当估算单体电压低于截止电压时，输出进入保护区；当估算单体电压高于过充电压时，进入过充预警。
- 电量按电流积分近似衰减，用于展示趋势而非精确 SOC 算法。

## 7. 交互与体验要求

- 页面优先桌面端可用，同时在移动端保持基本可读性。
- 用户无需跳转站外页面即可完成参数配置与结果查看。
- 视觉上应清晰区分输入区域、实时状态区域、配置区域和采样记录区域。
- 电池配置页面中的 BMS 与电芯分区要有明确标题、说明和摘要信息。
- 曲线配置区要支持查看关键点列表，避免只看到抽象参数。

## 8. 非功能要求

- 本地运行简单，默认通过 Vite 开发环境启动。
- 后端编译服务需预装 Emscripten (emcc)，首次运行需检查依赖是否就绪并给出提示。
- 代码结构保持轻量，便于后续替换为更真实的仿真模型。
- 允许在后续版本中接入图表组件和外部数据源。

## 9. 后续迭代方向

- 增加温度、单体压差、SOC 校准、均衡状态等 BMS 变量。
- 增加波形图或折线图，展示电压、电流和功率随时间变化。
- 增加典型场景模板，例如起步、爬坡、急加速、低电量返程。
- 增加故障注入，例如过流、欠压、采样异常、传感器漂移。
- 增加曲线点导入导出与 CSV/JSON 配置模板。
- 增加串并联拓扑和不同电芯化学体系选择。
- 增加地形可视化，将生成的海拔剖面与红绿灯位置渲染为路线图。
- 增加地形与电池仿真的联动：将坡度、停车等工况自动转化为功率需求输入。
- 增加预设种子方案库，提供典型城市/山区/平原等路况模板。
- 增加仿真数据导出功能，支持按秒级采样导出为 CSV/Excel 表格，包含完整状态记录。
- 增加自定义函数历史版本管理，支持回退至之前的算法版本。
- 增加多语言函数支持（如 Python、JavaScript），降低非 C 语言用户的使用门槛。
